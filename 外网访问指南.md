# MyBlog 外网访问指南

> 你的运营商：中国移动（辽宁大连）— 无公网IP，端口映射不可行  
> 当前方案：**Cloudflare Quick Tunnel**（免费临时隧道）

---

## 快速启动

### 1. 局域网访问（同WiFi设备）

博客绑定 `0.0.0.0:3000`，局域网设备直接访问：

```
http://192.168.1.39:3000
```

启动服务器：
```bat
双击 blogserver.bat  →  只启动博客服务器（局域网用）
```

### 2. 外网访问（不在同一WiFi）

```bat
双击 blogI.bat  →  启动博客 + Cloudflare Tunnel（外网用）
```

启动后会自动显示一个 **临时域名**，格式如：
```
https://gamma-unlikely-paint-cad.trycloudflare.com
```

> ⚠️ **每次启动域名都会变！** 记下当前域名分享给别人，下次重启后需要重新获取

---

## 外网地址每次都变怎么办？

Quick Tunnel 是免费的临时隧道，每次启动分配随机域名。有三种解决方案：

### 方案A：每次手动记录（当前方式）
- 启动 `blogI.bat`，看屏幕上的域名
- 每次分享给朋友时用新域名
- **优点**：零成本，零配置
- **缺点**：每次都变，不方便

### 方案B：注册 Cloudflare + 域名（固定域名，推荐）
1. 注册 Cloudflare 账号（免费） → https://dash.cloudflare.com
2. 买一个域名（便宜域名 .com 约 ¥60/年，.xyz 约 ¥7/年）
   - 可在 Cloudflare 直接买，也可在阿里云/Namesilo买后转入
3. 在 Cloudflare 创建 Named Tunnel：
   ```bash
   cloudflared tunnel create myblog
   cloudflared tunnel route dns myblog blog.你的域名.xyz
   ```
4. 启动固定隧道：
   ```bash
   cloudflared tunnel run --url http://localhost:3000 myblog
   ```
5. 访问 `https://blog.你的域名.xyz` — **固定域名，自带HTTPS！**

**优点**：域名固定，自动HTTPS，安全  
**缺点**：需买域名（几十块/年）

### 方案C：用免费子域名服务
- 使用 freedns.afraid.org 等免费子域名服务
- 但仍需配合 Named Tunnel，不如方案B直接

## 端口映射为什么不行？

你是中国移动用户，运营商给的IP是内网NAT地址（如 100.x.x.x），不是公网IP：
- 电信/联通用户打客服电话一般能申请到公网IP
- **移动基本不给公网IP**，多次申请也很难成功
- 所以端口映射这条路走不通，只能用隧道

## 安全建议 ⚠️

1. **不要用简单密码** — 一级密码至少8位，二级密码也要设
2. **定期更换密码** — 至少每3个月换一次
3. **Cloudflare Tunnel 自带HTTPS** — 数据传输已加密，比裸HTTP安全
4. **不要暴露管理接口** — 所有API都需要JWT认证
5. **定期备份** — 在设置页导出备份，存到 F:\WRO

## 问题排查

| 问题 | 解决方案 |
|---|---|
| Tunnel 启动失败 | 检查 cloudflared.exe 是否在 E:\myblogoc\myblog，VPN是否开着 |
| 域名访问502 | 博客服务器未启动，先重启 blogserver |
| VPN开着导致问题 | 关VPN再启动，或清除代理环境变量 |
| 局域网访问不了 | 检查电脑和手机是否在同一WiFi，IP是否变了 |
| 手机4G访问慢 | 正常现象，Tunnel经过海外中转 |
